  1. データ変換ロジック
    現行システムでは、JIDTKM21 ジョブ（ID: N_JIDTKM21_008）上で COBOL プログラムとユーティリティ（IKJEFT01／GALSV01）を一括実行し、内部処理がブラックボックス化されている。一方、次期システムでは、SQL_01～SQL_09 の各ステップを個別に実行し、段階的にデータ変換を行うモジュール化パイプラインを採用している。

  2. データ統合プロセス
    現行システムでは、単一ファイルベースの入力（FTP.S.MU.FSIMUMLF）を冒頭で取り込み、処理全体を通して一貫して利用する。次期システムでは、TMUBTZMISD10166 など複数ソースから段階的にデータを抽出・統合し、各 SQL ステップで必要なサブセットを結合する方式を採用している。
    証跡（必須）: 現行: FTP.S.MU.FSIMUMLF_flow.html / N_FTP_S_MU_FSIMUMLF; 次期: query_TMUBTMMISM10043_flow.html / TMUBTZMISD10166

  3. デバッグおよび検証手法
    現行システムは最終成果物出力後の検証（ブラックボックステスト）中心であり、中間処理の可視化や部分検証が困難。次期システムは各 SQL ステップ実行後にワークテーブルを介して中間データを検査できるため、ステップ単位のデバッグと検証が容易である。
    証跡（必須）: 現行: FTP.S.MU.FSIMUMLF_flow.html / N_JIDTKM21_010; 次期: query_TMUBTMMISM10043_flow.html / TMUWTMMISD10128
